# LogistiQ

India-first supply-chain SaaS for e-commerce and quick-commerce sellers: 31 free calculators, an **AI Demand Planner**, an **FBA Planner**, Sourcing & Procurement, and a TMS. Static site + Vercel serverless functions.

This README documents the **God-mode engine upgrades** (v11) layered on top of the existing product. Every upgrade is **additive and backend-only** — it changes no page visuals and no written/marketing copy. New capabilities are exposed as new fields in the API responses and as new serverless engines; the existing UI keeps working unchanged and can surface the new numbers when wired.

|---|---|---|

| `api/sourcing.mjs` | Sourcing & Procurement | Supplier scorecard (OTIF/lead-time/price/quality/fill), landed-cost comparison, reorder → PO (feeds inbound back to the planner). |
| `api/tms-rateshop.mjs` | TMS | **RTO-aware carrier rate-shop** (weight-slab + zone + COD), cost-per-delivered-order feeding unit economics. |
| `api/connectors/normalize.mjs` | Connectors | Canonical normalizer: Shopify + Amazon SP-API payloads → engine CSV. Add a marketplace = one normalizer. |
| `api/connect/shopify.mjs` | Connectors | **Shopify** OAuth install/callback + HMAC verify + Admin-API pull → normalize (secrets from env). |
| `api/connect/amazon.mjs` | Connectors | **Amazon SP-API** LWA token → Orders + FBA Inventory pull → normalize (official API, no scraping). |
| `api/history.mjs` + `js/liq-history.js` | Saved history | Merge to lengthen the series + **forecast-vs-actual** accuracy badge, persisted in the seller's existing Firestore (no new secrets). |

The AI Demand Planner engine (`api/forecast.js`) now returns, additively: per-SKU `econ` + `forecastQuantiles`, and top-level `buyPlan`, `purchaseOrders`, and `profitLeaks`.

## Architecture notes

- **Module system is mixed, by design.** `forecast.js`, `copilot.mjs` and all new engines use **ESM**; `distance.js`, `sr-rates.js`, `sr-track.js`, `tms-ai.js`, `_sr.js` use **CommonJS**. There is intentionally **no root `package.json`** — Vercel detects module type per file. New engines ship as **`.mjs`** so they are unambiguously ESM everywhere (Vercel + local Node) without a `package.json`, and never affect the CommonJS functions.
- **Self-contained engines.** `fba-forecast.mjs`, `sourcing.mjs`, `tms-rateshop.mjs` import at most `probabilistic.mjs`, so they run in plain Node with zero setup. `forecast.js` imports `econ.mjs` / `probabilistic.mjs` / `buyplan.mjs`.
- **Privacy model preserved.** Gemini (`gemini-2.5-flash`) writes *narrative insights only*, from computed summary numbers — never raw uploaded data. Forecast figures are always computed, never generated by the LLM.

## Environment variables

Set in Vercel Project Settings → Environment Variables (never commit secrets):

```
GEMINI_API_KEY=<your key>        # /api/forecast + /api/fba-forecast narrative insights only

# Connectors (optional — only if you enable Shopify / Amazon sync). See CONNECTORS.md.
SHOPIFY_API_KEY=...
SHOPIFY_API_SECRET=...           # never commit
SHOPIFY_SCOPES=read_orders,read_products,read_inventory
APP_URL=https://YOUR_DOMAIN
LWA_CLIENT_ID=...
LWA_CLIENT_SECRET=...            # never commit
SPAPI_REFRESH_TOKEN=...          # never commit
SPAPI_HOST=sellingpartnerapi-eu.amazon.com
```

Both planners fall back to deterministic insights when `GEMINI_API_KEY` is absent, so they work without it. Connectors are optional; the planners work upload-only until you configure them. **Saved history/accuracy needs no secrets** — it reuses the seller's existing Firestore account.

## Fee / economics assumptions

All commission, fee, storage, RTO and carrier figures in `econ.mjs`, `fba-forecast.mjs` and `tms-rateshop.mjs` are **indicative India defaults** — directional planning inputs, seller-overridable, and superseded by real values when present in the uploaded reports. They are not billed rates. Always validate against your ERP/WMS/marketplace before purchasing.

## Running the tests

```bash

node test/fba.test.mjs         # FBA forecasting engine           (20 assertions)
node test/tools.test.mjs       # sourcing + TMS engines           (20 assertions)
node test/connectors.test.mjs  # normalizers + HMAC + history     (24 assertions)
```

## Deploy

Deploys as-is on Vercel (static files + `/api` serverless functions). `vercel.json` sets a 30s max duration for `api/forecast.js` and `api/fba-forecast.mjs`.
